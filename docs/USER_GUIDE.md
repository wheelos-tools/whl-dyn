## 环境准备

### 系统要求

- Linux 系统 (Ubuntu 20.04+)
- Docker 环境
- Apollo 自动驾驶平台
- Python 3.7+

### 步骤1：克隆代码到 Apollo 目录

```bash
# 在容器外执行
cd /path/to/apollo/modules/tools/
git clone https://github.com/wheelos-tools/whl-dyn.git whl-dyn
```

或如果已有代码：

```bash
# 确保代码位于正确目录
ls /apollo/modules/tools/whl-dyn
```

### 步骤2：进入 Apollo 容器

```bash
# 进入 Apollo 开发容器
./docker/scripts/dev_start.sh
./docker/scripts/dev_into.sh
```

### 步骤3：升级 pip

```bash
# 在容器内执行
pip install --upgrade pip

# 或使用国内镜像加速
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple --upgrade pip
```

### 步骤4：安装

```bash
# 在容器内，进入工具目录
cd /apollo/modules/tools/whl-dyn

# 安装包（包含所有依赖，并创建 whl-dyn 可执行命令）
pip install -e .
```

## 快速开始

### 启动应用

```bash
# 在容器内执行
whl-dyn
```

启动后，在浏览器中访问：`http://localhost:8501`

**命令行参数**（容器内）：

```bash
# 指定端口
whl-dyn --server.port 8502

# 允许外部访问
whl-dyn --server.address 0.0.0.0

# 开发模式（代码变化自动刷新）
whl-dyn --server.runOnSave true

# 组合使用
whl-dyn --server.port 8502 --server.address 0.0.0.0
```

## 功能模块详解

### 📋 ① 生成计划

**功能**：配置测试参数并生成标定测试计划（容器内操作）

#### 界面布局

```
┌─────────────────────────────────┬──────────────┐
│  参数设置                        │  操作         │
│  ┌─────────────────────────┐    │              │
│  │ 油门范围: [====|====]   │    │  计划文件路径 │
│  │ 油门步数: [5]            │    │  [生成计划]  │
│  ├─────────────────────────┤    │              │
│  │ 刹车范围: [====|====]   │    │              │
│  │ 刹车步数: [5]            │    │              │
│  ├─────────────────────────┤    │              │
│  │ 速度目标: [3.0]         │    │              │
│  │ 默认油门: [80%]         │    │              │
│  │ 默认刹车: [30%]         │    │              │
│  │ 保持时间: [500ms]       │    │              │
│  └─────────────────────────┘    │              │
└─────────────────────────────────┴──────────────┘

┌─────────────────────────────────────────────────┐
│  视图模式: [表格] [源码]                          │
│  ┌───────────────────────────────────────────┐  │
│  │ 测试用例表格                              │  │
│  │ ┌────────────────────────────────────┐   │  │
│  │ │ case_name | description | steps    │   │  │
│  │ │ throttle_20_to_3mps | ...        │   │  │
│  │ │ brake_50_from_5mps | ...         │   │  │
│  │ └────────────────────────────────────┘   │  │
│  └───────────────────────────────────────────┘  │
└─────────────────────────────────────────────────┘
```

#### 参数说明

| 参数 | 类型 | 范围 | 默认值 | 说明 |
|------|------|------|--------|------|
| 油门范围 | 滑块 | 0-100% | 0-80% | 油门测试的最小和最大值 |
| 油门步数 | 数字 | 1-30 | 5 | 生成测试点的数量 |
| 刹车范围 | 滑块 | 0-100% | 0-80% | 刹车测试的最小和最大值 |
| 刹车步数 | 数字 | 1-30 | 5 | 生成测试点的数量 |
| 速度目标 | 文本 | - | "3.0" | 目标速度(m/s)，逗号分隔 |
| 默认油门 | 数字 | 0-100% | 80% | 刹车测试前加速用 |
| 默认刹车 | 数字 | 0-100% | 30% | 测试结束停止用 |
| 保持时间 | 数字 | 0-5000ms | 500ms | 达到目标后的保持时间 |

#### 操作步骤

1. 设置测试参数
2. 点击"生成计划"按钮
3. 在下方查看生成的测试用例（表格或源码视图）
4. 测试计划保存到容器内：`/apollo/modules/tools/whl-dyn/calibration_plan.yaml`

### 🚗 ② 数据采集

**功能**：执行测试计划并实时监控采集过程（容器内操作）

#### 界面布局

```
┌─────────────────────────────────────────────────┐
│  计划文件: [calibration_plan.yaml]              │
│  输出目录: [/apollo/modules/tools/whl-dyn/...]  │
└─────────────────────────────────────────────────┘

┌──────────────────────┬──────────────────────────┐
│  用例选择与进度       │  当前状态与控制           │
│  ┌────────────────┐  │  ┌────────────────────┐  │
│  │◀ [选择用例] ▶ │  │  │ 选中用例:          │  │
│  ├────────────────┤  │  │ 组选择: [第1组▼]   │  │
│  │ 执行进度列表:   │  │  ├────────────────────┤  │
│  │ ┌────────────┐ │  │  │ ▶开始 ⏹停止      │  │
│  │ │throttle_20 │2组││  │ │ ↺重试 🗑清除      │  │
│  │ │      ✓   │156││  │  │ [✓通过] [✗打回]   │  │
│  │ ├────────────┤ │  │  ├────────────────────┤  │

│  │ └────────────┘ │  │  │ $ collecting data   │  │
│  └────────────────┘  │  │ $ done              │  │
│                      │  └────────────────────┘  │
└──────────────────────┴──────────────────────────┘
```

#### 左侧 - 用例选择与进度

**用例选择器**：
- `◀` 按钮：选择上一个用例
- 下拉框：直接选择用例
- `▶` 按钮：选择下一个用例

**执行进度列表**：

| 列 | 说明 |
|----|------|
| 用例名 | 测试用例名称（自动缩写） |
| 组数 | 已采集的数据组数 |
| 状态 | 当前状态图标 |
| 行数 | 所有组的总数据行数 |

**状态说明**：

| 图标 | 状态 | 说明 |
|------|------|------|
| ⚪ | pending | 待执行 |
| 🟡 | collecting | 采集中 |
| ✓ | completed | 已完成/已通过 |
| ✗ | rejected | 已打回/数据不合格 |

#### 右侧 - 当前状态与控制

**组选择器**：
- 查看不同组的数据（第1组、第2组...）
- 仅当有多个数据组时可用

**操作按钮**：

```
┌────────┬────────┬────────┬────────┬────────┬────────┐
│ ▶ 开始 │ ⏹ 停止 │ ↺ 重试 │ 🗑 清除 │ ✓ 通过 │ ✗ 打回 │
└────────┴────────┴────────┴────────┴────────┴────────┘
```

| 按钮 | 功能 | 启用条件 |
|------|------|----------|
| ▶ 开始 | 开始采集 | 查看最后一组且未运行 |
| ⏹ 停止 | 停止采集 | 正在运行 |
| ↺ 重试 | 删除最后一组并重新采集 | 有数据且未运行 |
| 🗑 清除 | 删除当前选中的组 | 有数据且未在等待确认 |
| ✓ 通过 | 确认数据合格 | 等待确认中 |
| ✗ 打回 | 数据不合格 | 等待确认中 |

**质量指标显示**：

- **采集行数**：当前数据文件的总行数
- **速度跨度**：速度变化范围 (m/s)
- **指令跨度**：控制指令变化范围 (%)

#### 采集流程

1. **选择用例**：在左侧选择要执行的用例
2. **开始采集**：点击"▶ 开始"
3. **监控过程**：观察实时日志和状态
4. **采集完成**：程序自动停止或手动停止
5. **确认数据**：
   - 合格 → 点击"✓ 通过"
   - 不合格 → 点击"✗ 打回"或"↺ 重试"

#### 数据存储位置

采集数据保存在容器内：
```
/apollo/modules/tools/whl-dyn/calibration_data_logs/
├── throttle_20_to_3mps_0.csv
├── throttle_20_to_3mps_1.csv
├── brake_50_from_5mps_0.csv
└── ...
```

#### 多组数据采集

对于同一用例采集多组数据：

```
完成第1组 → ✓通过 → 选择同一用例 → ▶开始 → 完成第2组 → ✓通过
```

## 常见问题

### Q1: 如何进入 Apollo 容器？

```bash
# 在宿主机执行
./docker/scripts/dev_start.sh
./docker/scripts/dev_into.sh

# 或使用一条命令
./docker/scripts/dev_start.sh && ./docker/scripts/dev_into.sh
```

### Q2: pip 安装依赖很慢怎么办？

```bash
# 在容器内使用国内镜像源
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt

# 或永久配置镜像源
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q3: 启动后无法访问？

**检查清单**：
1. 确认在 Apollo 容器内运行
2. 检查端口 8501 是否被占用
3. 尝试使用 `--server.address 0.0.0.0` 参数

```bash
# 在容器内查看 Streamlit 进程
ps aux | grep streamlit

# 查看端口占用
netstat -tlnp | grep 8501
```

### Q4: 数据采集程序启动失败？

**可能原因**：
1. CyberRT 未启动
2. 计划文件路径错误
3. 输出目录无写入权限
4. 车辆未连接

**解决方法**：
```bash
# 在容器内检查 CyberRT
ps aux | grep cyber

# 检查文件权限
ls -la /apollo/modules/tools/whl-dyn/calibration_data_logs

# 查看实时日志了解详细错误
```

### Q5: 如何判断数据质量？

**关键指标**：
1. **采集行数**：建议 > 100
2. **速度跨度**：应覆盖目标速度
3. **指令跨度**：应符合测试设置
4. **单调性**：必须为0违反（硬性要求）

### Q6: "通过"和"打回"有什么区别？

| 操作 | 结果 | 状态变化 |
|------|------|----------|
| ✓ 通过 | 数据保留 | completed → 切换到下一用例 |
| ✗ 打回 | 数据删除 | rejected → 保持当前用例 |

### Q7: 如何采集多组数据？

```
选择用例 → ▶开始 → 完成采集 → ✓通过
→ 再次选择同一用例 → ▶开始 → 完成第2组 → ✓通过
```

### Q8: 分析页面没有数据？

**检查项**：
1. 数据目录路径是否正确（容器内路径）
2. 目录中是否有 CSV 文件
3. 尝试清除缓存：侧边栏 → 🔄 清除缓存
4. 调整参数后重新加载

### Q9: 数据在哪里？

**采集数据**（容器内）：
```
/apollo/modules/tools/whl-dyn/calibration_data_logs/
```

**分析结果**（容器内）：
```
/apollo/modules/tools/whl-dyn/calibration_results/
```

### Q10: 如何将数据复制到容器外？

```bash
# 在宿主机执行
docker cp <container_id>:/apollo/modules/tools/whl-dyn/calibration_data_logs ./local_backup

# 或使用 docker cp
docker cp apollo_dev_container:/apollo/modules/tools/whl-dyn/calibration_data_logs ./
```

### Q11: 如何提高标定精度？

**建议**：
1. 每个测试点采集多组数据
2. 选择平坦的测试场地
3. 避免大风天气
4. 调整延迟补偿参数
5. 增加数据采集时间

### B. 数据文件格式

**CSV 文件列**：

```csv
time,speed_mps,ins_speed_mps,imu_accel_y,driving_mode,actual_gear,throttle_pct,brake_pct,ctl_throttle,ctl_brake
```

| 列名 | 类型 | 说明 | 单位 |
|------|------|------|------|
| time | float | 时间戳 | s |
| speed_mps | float | 底盘速度 | m/s |
| ins_speed_mps | float | 定位速度 | m/s |
| imu_accel_y | float | IMU纵向加速度 | m/s² |
| driving_mode | int | 驾驶模式 | - |
| actual_gear | int | 实际档位 | - |
| throttle_pct | float | 实际油门开度 | % |
| brake_pct | float | 实际刹车压力 | % |
| ctl_throttle | float | 控制油门指令 | % |
| ctl_brake | float | 控制刹车指令 | % |

### C. YAML 计划文件格式

```yaml
- case_name: throttle_20_to_3mps
  description: Accelerate with 20% throttle, target >3m/s, then brake to stop.
  steps:
    - command:
        throttle: 20.0
        brake: 0.0
      trigger:
        type: speed_greater_than
        value: 3.0
      timeout_sec: 30.0
      hold_duration_ms: 500
    - command:
        throttle: 0.0
        brake: 30.0
      trigger:
        type: speed_less_than
        value: 0.1
      timeout_sec: 30.0
```

### D. 术语表

| 术语 | 英文 | 说明 |
|------|------|------|
| 死区 | Deadzone | 不产生响应的最小指令值 |
| 线性度 | Linearity | 响应与指令的线性相关程度 |
| 平滑度 | Smoothness | 响应曲线的平滑程度 |
| 单调性 | Monotonicity | 指令增加时响应是否单调增加 |
| LOF | Local Outlier Factor | 局部异常因子，用于异常值检测 |
| 延迟补偿 | Latency Compensation | 补偿控制到响应的时间延迟 |
| 稳定性窗口 | Stability Window | 过滤瞬态数据的时间窗口 |
